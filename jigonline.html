JigLite

In the past, applications like flash made it easy to manage animations intended for web, using an actual timeline interface. The concept behind these crazy names is to mimick that timeline, but in a way that you can keep in you head. So each of these crazy names have a relationship with one another.

Enter TimelineLite, an intuitve animation platform. TimelineLite is a very powerful platform which you can learn more about at greensock.com. By using TimelineLite as Jigs core functionality, so this means you can control zigs and ziggles with TimelineLites native functions like 'play()' and 'totalDuration()'

 jig();

A jig is the object that contains tha animation for a particular DOM element. In theory ( since Jig is still in alpha) You can have as many jigs as you want for a DOM element, but it is easier just to append all your animation to the DOM element in one chain. The jig exhists so the developer can simply inspect the jig's object and have all the information and control needed of the jigs DOM elements animations. SO a jig holds the timeline that is related to the jigs DOM element and then you can append that timeline to any other timeline you wish, using TimelineLite.

 wiggle();

Zigs are timelines that control a preset animation like 'wiggle', each instance of 'wiggle has it's own Zig. Ziggles are the tiny pieces of a zig animation, if your zig animation is part of a jig that is targeting an arry of items, like a class, then your zig will have many ziggles, each ziggle related to each item in your jigs array of DOM elements, however the zig will still be the primary cpontroller of all these ziggles.

So in conclusion, these quick little words exhist to distiguish between a potentially confusing system of anonymous instances of timelines created by TimelineLite. They just make it more realtable by giving each anonaymous timeline a standardized label.
